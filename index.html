What is Software Engineering?

Software Engineering is the systematic process of designing, developing, testing, and maintaining software using techniques like requirements analysis, design, testing, and maintenance. It ensures software is high-quality, reliable, and maintainable, especially for large software systems. Software Engineering improves efficiency, quality, and budget management, helping deliver software on time, within budget, and meeting all requirements It continuously adopting new tools and methodologies to handle complexity and evolving technologies.

Key Principles

  • Modularity
  • Abstraction
  • Reusability
  • Maintenance
  • Testing
  • Design Patterns
  • Agile Methodologies

Software Development Life Cycle (SDLC)

The SDLC outlines the process used by software industries to design, develop, and test high-quality software. For the Nykaa application, an Agile methodology was simulated to adapt to changing market requirements in the beauty industry.

Stages of the Software Development Life Cycle:

  • Stage 1: Planning & Feasibility Analysis
  • Stage 2: Requirement Specification (SRS)
  • Stage 3: System Design
  • Stage 4: Development (Coding)
  • Stage 5: Testing
  • Stage 6: Deployment

Requirement Analysis

Defining the Scope

Requirement analysis is the process of defining user expectations for a new or modified product. For an e-commerce platform like Nykaa, this involves understanding both the customer's journey (browsing, cart, checkout) and the vendor's administrative needs.

Functional Requirements

  • User registration and authentication (Email, OTP).
  • Product catalog browsing with advanced filters.
  • Shopping cart management and secure checkout.
  • Order tracking and purchase history.

Non-Functional Requirements

  • Performance: Page load under 2 seconds.
  • Scalability: Handle 10,000+ concurrent users during sales.
  • Security: PCI-DSS compliance for payments.
  • Usability: Intuitive mobile-first design.

UML / Diagrams

Unified Modeling Language (UML) diagrams help visualize the architecture, design, and implementation of complex software systems. Below are the static and dynamic models for the application.

Static Diagrams

1. Use Case Diagram

A Use Case Diagram is a visual way to show how users (actors) interact with a system and what functions (use cases) the system provides. It helps understand the system’s behavior from the user’s perspective.

  • Represents actors (users or external systems) and use cases (system functionalities).
  • Shows the relationships between users and system features.
  • Helps define system scope, requirements, and interactions clearly.

2. Activity Diagram

Activity diagrams show the steps involved in how a system works, helping us understand the flow of control. They display the order in which activities happen and whether they occur one after the other (sequential) or at the same time (concurrent).

  • An activity diagram starts from an initial point and ends at a final point, showing different decision paths along the way.
  • They are often used in business and process modeling to show how a system behaves over time.

Dynamic Diagrams

1. Class Diagram

A UML class diagram shows the structure of a system by displaying its classes, their attributes, methods, and the relationships between them. It helps the team understand how the system is organized and how components interact.

  • Represents classes, attributes, methods, and relationships.
  • Helps communicate and document the software structure.
  • Makes it easier for developers and designers to understand the system.

2. Component Diagram

Component diagram shows how the components of a system are arranged and relate to one another. They are widely used in system design to promote modularity, enhance understanding of system architecture.

  • System components are modular units that offer a set of interfaces and encapsulate implementation.
  • These diagrams illustrate how components are wired together to form larger systems, detailing their dependencies and interactions.

3. Deployment Diagram

A Deployment Diagram shows how the software design turns into the actual physical system where the software will run. They show where software components are placed on hardware devices and shows how they connect with each other. This diagram helps visualize how the software will operate across different devices.

Testing

Test Case Analysis

Software testing ensures the application meets all requirements and functions perfectly under various conditions. For an e-commerce platform, rigor in testing is crucial to prevent revenue loss and protect user data.

Testing Strategies Implemented:

  • Unit Testing (Individual module validation)
  • Integration Testing (API to DB communication)
  • System Testing (End-to-end user flow)
  • Stress Testing (High traffic simulation)
